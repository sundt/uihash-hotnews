# Change: Update Category Settings UX

## Why
栏目设置当前不够人性化：
- 默认栏目列表占用空间较大，用户自定义时需要频繁上下滚动。
- 平台选择区域需要滚动，勾选效率低。

目标是在**不改变任何功能与既有操作语义**（新增/编辑/删除/排序/隐藏/恢复默认/保存/完成）的前提下，优化布局与交互，让用户更快完成配置。

## What Changes
- 重设计栏目设置弹窗布局：
  - 将“默认栏目”和“自定义栏目”在视觉上分区（或标签页/折叠区），默认栏目默认折叠或缩小占位。
  - 自定义相关（新增/编辑）区域更突出，减少滚动。
- 优化平台选择 UI：
  - 平台列表改为多列网格（自适应宽度），尽量在一屏内展示更多平台（方案 A：优先一屏展示，极端情况下允许少量滚动，不默认全屏）。
  - 提供平台快速筛选（搜索框）与批量操作（全选/全不选/清空）。
  - 保持拖拽排序能力不变（平台顺序仍可调整）。
- 操作语义保持不变：
  - “保存”仍保存当前编辑。
  - “完成”仍退出栏目设置并应用配置。
  - “恢复默认”仍清空配置并回到默认。

## Impact
- Affected specs:
  - `viewer-categories`（仅 UX / UI 呈现层面调整，不改变需求语义）
- Affected code:
  - `trendradar/web/templates/viewer.html`（弹窗 HTML/CSS/JS）
  - `tests/e2e/category-settings.spec.ts`（可能需要更新选择器；补充 UX 相关 E2E 断言）

## Out of Scope
- 不新增新的功能点（例如：权限、多端同步、服务端存储等）。
- 不改变任何已存在的配置结构与 localStorage key。

## Risks
- UI 结构调整可能导致 Playwright 选择器失效，需要同步更新测试并确保 `npm test` 全部通过。

## Context
栏目设置目前在自定义栏目时交互效率较低：默认栏目列表占用空间大、平台选择需滚动。

本变更仅调整 UI/UX 呈现，不改变任何功能语义与数据结构（localStorage key/结构不变）。

## Goals / Non-Goals
- Goals:
  - 提升自定义栏目创建/编辑效率
  - 方案 A：平台选择尽量一屏展示，极端情况下允许少量滚动
  - 保持现有操作语义不变（保存/完成/恢复默认/拖拽排序等）
  - 提升 Playwright 选择器稳定性（优先使用 id/data-*）
- Non-Goals:
  - 不新增服务端存储/同步
  - 不改变配置 schema 与 key

## Decisions
- 弹窗布局
  - “默认栏目”与“自定义栏目”分区展示（折叠/缩小默认栏目区），避免默认栏目占用过多空间。
  - 编辑面板优先置顶或更靠前，减少用户滚动距离。
- 平台选择
  - 使用 CSS Grid 多列布局（`grid-template-columns: repeat(auto-fit, minmax(...))`）
  - 增加搜索框过滤平台条目
  - 增加批量操作按钮（全选/全不选/清空）
  - 仍支持拖拽排序（拖拽在过滤视图下行为需明确：通常仅对当前可见列表排序或退出过滤后排序）

## Risks / Trade-offs
- UI 结构变动会影响现有 Playwright 选择器
  - 缓解：为关键元素补 `id` / `data-testid`（如允许），并更新 Page Object。
- 平台过多时仍可能出现滚动
  - 符合方案 A 约束：尽量一屏，极端情况允许少量滚动。

## Open Questions
- 平台搜索过滤时的拖拽排序策略（建议：搜索开启时禁用拖拽，并提示“清空搜索后可拖拽排序”）。
